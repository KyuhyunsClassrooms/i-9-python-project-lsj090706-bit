# 프로젝트 계획서

## 1. 프로젝트 주제

내가 만들 프로그램의 주제:나의 현재 예산과 취향(맛)에 따른 맞춤형 디저트 추천 및 가상 주문 프로그램

예) 기분과 시간에 따른 활동 추천기

---

## 2. 문제 상황

이 프로그램이 해결하려는 문제를 설명하세요.

예) 사용자가 현재 기분과 사용할 수 있는 시간을 입력하면, 조건에 맞는 활동을 추천해 주고 싶다.

내 설명:학생들이 매점이나 카페에 갔을 때, 자신이 가진 돈(예산) 안에서 무엇을 먹을지 고민하거나 결정 장애를 겪는 경우가 많다. 이 프로그램은 사용자가 현재 가지고 있는 금액과 선호하는 맛(달콤한 맛, 상큼한 맛, 짭짤한 맛 등)을 입력하면, 조건에 맞는 디저트를 추천해 주고 가상으로 주문 금액을 계산해 주어 합리적인 소비 선택을 도와준다.


---

## 3. 입력-처리-출력 구조

### 입력

사용자가 입력하거나 프로그램이 사용할 데이터는 무엇인가?

- 사용자의 현재 예산 (정수형 금액, 예: 5000)
- 사용자가 원하는 맛 취향 (문자열, 예: "달콤함", "상큼함", "짭짤함")
- 구매하고 싶은 디저트의 번호 및 수량 (메뉴 선택 시 입력)

### 처리

입력된 데이터를 어떤 규칙이나 알고리즘으로 처리할 것인가?

- 전체 디저트 데이터(2차원 리스트)를 순회하며, 사용자의 예산 이하이면서 맛 취향이 일치하는 메뉴만 필터링한다.
- 필터링된 메뉴가 있다면 화면에 추천 리스트로 보여준다.
- 사용자가 메뉴를 선택하면 가격 * 수량을 계산하여 총금액을 구하고, 잔액이 부족하지 않은지 검사한다.



### 출력

프로그램이 최종적으로 무엇을 출력하는가?

- 조건(예산, 취향)에 맞는 맞춤형 디저트 추천 메뉴판
- 가상 주문 결과 (선택한 메뉴, 총 주문 금액, 구매 후 남은 잔액)

## 6. 사용할 제어구조

### 조건문을 사용할 부분

예) 사용자의 예산보다 가격이 낮은 항목만 추천한다.

내 계획:2차원 리스트의 각 항목을 확인할 때, 디저트 가격 <= 사용자 예산 이고 디저트 맛 == 사용자 취향 인지 비교할 때 if 조건문을 사용한다.

주문할 때 총 주문 금액 > 현재 예산 이면 "잔액이 부족합니다"를 출력하고, 그렇지 않으면 주문을 확정하는 if-else 구조를 사용한다.


### 반복문을 사용할 부분

예) 2차원 리스트의 모든 항목을 하나씩 검사한다.

내 계획:for문을 사용하여 desserts 리스트의 처음부터 끝까지 한 행씩 접근하여 조건에 맞는 메뉴가 있는지 전수 조사(검색)한다.

사용자가 올바른 메뉴 번호를 입력할 때까지 계속해서 입력을 받기 위해 while문을 사용한다.
